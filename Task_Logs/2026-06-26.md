## 下午追加 #3：4 個工作流問題處理

### 動機
走查 App.jsx → DetailModal 後，發現多項架構性缺陷。

### 問題與解法

#### 問題 1：30 天價格曲線 + 分頁處理
- **現況**：`fetchPrices()` 只抓近 4 天，DetailModal `slice(-7)` 永遠看不到 7 天
- **解法**：
  - `fetchPrices({ days = 30 })` 預設 30 天範圍
  - 切成不重疊的 7 天/段，5 段並行抓取
  - `Map` 去重（`日期|作物|市場` 三元組）避免接縫重複
  - `MAX_DAYS=30` 為硬上限，防止惡意呼叫

#### 問題 2：DetailModal cropName 嚴格相等
- **現況**：`甘藍-初秋` 與 `甘藍-改良尖` 視為兩個不同作物，點開 DetailModal 只看自己變體
- **解法**：
  - `aggregation.js` 在 crop 物件加 `mainName` 欄位（去掉變體後的主品項）
  - `DetailModal.jsx` 改用 `splitMarketName(r.作物名稱).main === mainName` 模糊比對
  - 同主品項的所有變體會彙整進趨勢圖

#### 問題 3：fetchPrices 沒有 timeout/retry
- **現況**：政府 API 偶爾慢/掛，會讓 spinner 永遠轉
- **解法**：
  - `AbortController` 15s timeout
  - 每段最多重試 1 次（MAX_RETRIES）
  - 失敗 console.warn，徹底失敗 console.error，仍降級回傳 `[]`

#### 問題 4：MARKET_ORDER 重複定義
- **現況**：`['台北二', '台中市', '高雄市']` 在 4 個檔案寫死
- **解法**：
  - `api.js` 改為 `export const MARKET_ORDER = Object.values(MARKET_IDS).map(m => m.name)` 推導
  - `DetailModal.jsx` 移除本地寫死，改 import
  - `ProductCard.jsx` 移除本地寫死，改 import

## 下午追加 #5：Cache 瘦身 + GitHub Pages 部署 + App 自更新

### A. Cache 瘦身

**問題**：原始 cache 3.26 MB 偏大，App 啟動下載 ~1-2 秒。

**解法**：移除不必要欄位 + 改用單行 JSON 寫入。
- 移除欄位：`交易量`（無用）、`種類代碼`（已過濾）、`市場代號`（市場名稱已涵蓋）
- 改用 `JSON.stringify(result)` 而非 `JSON.stringify(result, null, 2)`（省縮排空白）
- dedup key 從「市場代號」改為「市場名稱」（已同步 daily_cache.cjs 與 api.js）

**結果**：3.26 MB → **1.68 MB（省 48.5%）**
